五、模块实现

-——— 商品管理模块 -———

(一) 品牌管理后端
(1) 品牌列表

  1. 需求分析:
    实现对品牌的基本操作(增删改查)
  2. 表结构分析
  3. 代码实现
    1. 创建实体类
      在 pojo 工程下创建实体类
      分别用 @Table(在类上) 和 @Id(在属性id上) 标签标明数据库表和对应的主键
    2. 在服务层模块下创建数据访问层
      创建一个dao接口，并且该接口继承 Mapper<> ,其泛型为对应的实体类
    3. 做服务接口
      在业务接口的公共模块下写品牌的 service 接口
    4. 在服务层模块下写 service 接口的实现类
      调用数据访问层
      在类上添加 @Service 注解 (注意添加 dubbo下的@Service注解)
      import com.alibaba.dubbo.config.annotation.Service;
    5. 写 web 层来调用服务层的方法
      在 web 层模块下新建一个 controller 控制器的类
      在类上添加 @RestController 和 @RequestMapping(“/brand”)两个注解
      (@Controller和@RestController的区别？)
    6. 测试
      1.启动 zookeeper 服务 (zookeeper相关使用参照此处)
      1. 由于前面修改了公共接口模块和公共实体类模块这两个工程，所以需要安装这两个工程(注意公共模块发生变化时都要进行安装一下)
        在IDEA右侧 Maven Projects 下分别点开以上两个工程的 Lifecycle, 选择 install 双击执行即可
      2. 运行程序
        首先运行服务层
        点开 Plugins 双击 tomcat 下的 run 即可
        然后运行 web 层
        点开 Plugins 双击 tomcat 下的 run 即可
      3. 观察测试结果
        在浏览器中输入http://localhost:9101/brand/findAll.do
  4. 踩过的坑
    当启动 tomcat 时，（一般都是重新执行 install 安装过的服务层公共模块工程）在启动过程中会报类似如下错误:

解决方法:
在对子模块进行 clean 和install 前先对父工程进行clean 和install操作。

总结: 在父项目下有的子项目在首次运行clean 和install前应该先运行父项目的clean和install

(2) 品牌分页

  1. 在公共实体模块下创建 entity 包, 在该包下创建 PageResult 类。
    • pojo 包下放和数据库相连的实体类
    • entity 包下放具有通用性的封装实体类
  2. 在公共接口模块下的品牌的 service 接口里添加方法返回分页数据
  3. 在对应的服务层模块下的service 实现类中实现上面接口中添加的方法
  4. 到web 层调用服务层的方法，在品牌管理的 controller 控制器的类中添加分页查询方法，并在该方法上添加 @GetMapping(“/findPage”) 注解，此时只能用 get 方式提交请求
  5. 品牌分页测试
    安装修改过的公共模块(执行 install 操作)
    启动服务器(先确保 zookeeper 服务开启，然后先启动服务层，再启动 web 层)
    浏览器中输入 localhost:9101/brand/findPage.do?page=1&size=10(表示查询的第一页数据，每页显示10条)
